# Development Rules

Use:
- Next.js App Router
- TypeScript only
- Tailwind CSS
- shadcn/ui
- Mobile-first design
- Reusable components
- Server Components by default

Branding Rules:
- Use existing logo from /public folder
- Do not create a new logo
- Do not use robot dashboard wording
- Do not assume the club has robots
- Focus on IoT equipment and electronics components

Rules:
- No inline styles
- No plain CSS files
- Use Tailwind classes
- Use TypeScript types/interfaces
- Keep components modular
- Use clean folder structure
- Use async/await
- Validate all forms
- Use responsive layouts

Preferred UI:
- Minimal luxury design
- Rounded cards
- Soft shadows
- Spacious layout
- Elegant typography
- Premium university inventory system
- Clean tables
- Consistent spacing

Avoid:
- Neon colors
- Heavy gradients
- Complex animations
- Over-designed UI
- Massive components
- Cartoon style
- Robot dashboard UI
